# APM Spec

## Overview

ClipboardSync is a cross-platform, cross-device clipboard synchronization tool for local area networks. It enables real-time sharing of clipboard content (text, images, files) between macOS and Windows machines using a peer-to-peer architecture with no central server. The tool prioritizes security through a pairing mechanism and encrypted transport, stores clipboard history with configurable retention, and provides a native GUI built with Wails (Go + Web frontend). Success criteria: two or more devices on the same LAN can pair, synchronize clipboard content in real-time, and browse/search history through a desktop application.

## Architecture

### Network Topology

- **Model:** Pure P2P, no central server or coordinator
- **Discovery:** mDNS (multicast DNS) for automatic LAN device discovery
- **Transport:** TLS 1.3 encrypted connections for all data transfer
- **Connection model:** Each device maintains connections to all paired devices (mesh topology)
- **Scope:** LAN only — devices must be on the same subnet to discover each other

### Security Model

- **Pairing:** 6-digit numeric code, generated by one device, entered on the other
- **Code expiry:** Short-lived (e.g., 60 seconds) to limit exposure
- **Trust model:** Once paired, devices exchange and store cryptographic material for persistent authentication
- **Access control:** Only paired devices can request clipboard content or push updates
- **Transport encryption:** All clipboard data and control messages encrypted via TLS
- **No plaintext on wire:** Even within a trusted LAN, no clipboard content is transmitted unencrypted

### Clipboard Monitoring

- **Primary:** System event-driven notifications (macOS NSPasteboard, Windows clipboard listener)
- **Fallback:** 1000ms polling with content hash comparison for systems where events are unreliable
- **Resource target:** Minimal CPU and memory footprint when idle

### Data Model

**Clipboard Entry:**
- Unique ID (UUID)
- Content type (text, image, file)
- Content hash (SHA-256, for deduplication)
- Payload (text content or file path reference)
- Source device ID
- Timestamp
- Size

**Device:**
- Device ID (generated on first run)
- Display name (auto-generated, user-editable)
- Pairing status
- Last seen timestamp
- Public key material

**History:**
- SQLite database per device
- LRU eviction: max entries configurable, default 1000
- Searchable by content (text entries) and metadata

### Sync Protocol

- **Push model:** When clipboard changes, the source device pushes the new entry to all connected peers
- **Deduplication:** Content hash comparison — identical content (regardless of source) stored as one entry
- **Conflict resolution:** All versions preserved — if two devices copy different content simultaneously, both entries appear in history
- **Transfer limits:** Single item max 50MB
- **Large transfers:** Background silent transfer with internal progress tracking

### Device Management

- **Naming:** Auto-generated on first run (e.g., "Device-A3F2"), user can rename
- **Multi-device:** Supports 3+ simultaneous connections
- **Status:** Real-time connection status visible in GUI

## User Interface

### Desktop Application (Wails)

- **Framework:** Wails v2 — Go backend, web frontend (HTML/CSS/JS), rendered as native window
- **Window:** Standard desktop window, not a browser

### Main Window

- **History view:** Scrollable list of clipboard entries with search bar
- **Entry display:** Preview of content (text snippet, image thumbnail, file name/icon)
- **Entry actions:** Copy to clipboard, delete from history
- **Search:** Full-text search across text history entries

### System Tray

- **Icon:** System tray icon indicating sync status (active/paused/disconnected)
- **Menu items:**
  - Open main window
  - Toggle sync on/off
  - View connected devices
  - Clear history
  - Quit

### Device Pairing UI

- **Initiate pairing:** Generate and display 6-digit code
- **Accept pairing:** Input field for entering code from another device
- **Device list:** Show all paired devices with name, status, last seen

## Distribution

- **Format:** Single executable per platform (`.app` for macOS, `.exe` for Windows)
- **Auto-start:** Not in MVP; designed to be configurable later
- **Dependencies:** Self-contained — no external runtime or framework installation required
